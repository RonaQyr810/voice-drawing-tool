# AI 语音绘图工具 — 设计文档

## 1. 项目概述

本项目是一款**纯语音控制**的 Web 绘图应用。用户无需使用鼠标或键盘即可完成绘图创作；所有交互均通过中文语音指令驱动。应用基于浏览器原生 **Web Speech API** 进行语音识别，使用 **HTML5 Canvas** 渲染图形，并通过自研**规则型命令解析器**将自然语言映射为绘图操作。

### 设计目标

| 目标 | 策略 |
|------|------|
| 指令理解准确性 | 关键词匹配 + 同义词表 + 正则提取参数 |
| 容错性 | 支持多种口语表达、去除语气词、识别结果多候选 |
| 响应延迟 | 连续识别模式、即时 interim 反馈、命令队列异步执行 |
| 复杂指令拆解 | 识别「然后/接着/再」等连接词，拆分为多步顺序执行 |

## 2. 系统架构

```
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  麦克风输入  │ ──► │  VoiceController │ ──► │  CommandParser  │
│ (Web Speech)│     │  连续识别/自动重启 │     │  文本→操作序列   │
└─────────────┘     └──────────────────┘     └────────┬────────┘
                                                      │
                                                      ▼
┌─────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  Canvas 画布 │ ◄── │  DrawingEngine   │ ◄── │   命令队列       │
│  图形渲染    │     │  图形绘制/历史    │     │  顺序执行        │
└─────────────┘     └──────────────────┘     └─────────────────┘
```

### 模块说明

- **VoiceController** (`js/voice-controller.js`)：封装 Web Speech API，支持连续监听、断线自动重启、interim/final 结果分流。
- **CommandParser** (`js/command-parser.js`)：将中文语音文本解析为结构化绘图指令，支持复合句拆解。
- **DrawingEngine** (`js/drawing-engine.js`)：Canvas 绑定的绘图引擎，管理画笔状态、图形绘制与撤销历史。
- **App** (`js/app.js`)：串联各模块，处理 UI 反馈，禁用鼠标/键盘对画布的干扰。

## 3. 计划支持的指令能力

### 3.1 语音控制类

| 指令 | 计划同义词 |
|------|-----------|
| 开始监听 | 开始、启动、开始录音、打开麦克风 |
| 停止监听 | 停止、暂停、关闭麦克风 |
| 撤销 | 撤回、上一步 |
| 清空画布 | 清除、重置画布 |
| 帮助 | 指令帮助、怎么用 |

### 3.2 基础图形类

| 图形 | 计划参数 |
|------|---------|
| 圆 | 颜色、位置、半径、空心/实心 |
| 矩形 | 颜色、位置、宽、高、空心/实心 |
| 三角形 | 颜色、位置、边长 |
| 椭圆 | 颜色、位置、宽、高 |
| 直线 | 颜色、起点、终点、线宽 |
| 多边形 | 边数（五/六边形）、半径 |
| 文字 | 内容、颜色、位置、字号 |

### 3.3 画笔与属性类

| 操作 | 计划表达方式 |
|------|-------------|
| 移动画笔 | 「移动到 300 400」 |
| 设置颜色 | 「设置颜色为红色」 |
| 设置线宽 | 「线宽 5」 |

### 3.4 位置表达

- 语义位置：中心、左上、右下、上方、左侧等
- 坐标位置：「在 300 400」「位置 100,200」
- 线段端点：「从左上到右下」

### 3.5 复合指令

- 连接词：然后、接着、再、之后、并且
- 示例：「画一个红色的圆然后画一个蓝色的矩形」

### 3.6 高级能力（计划中）

| 能力 | 说明 |
|------|------|
| 自由手绘 | 「从这里画到那里」连续路径 |
| 图形变换 | 移动、缩放、旋转已有图形 |
| 图层管理 | 选中、删除指定图形 |
| AI 语义增强 | 接入 LLM 理解更自由的描述 |
| 离线识别 | 本地 Whisper 等模型 |
| 语音合成反馈 | TTS 播报执行结果 |

## 4. 最终实现情况

### 4.1 已实现 ✅

| 类别 | 实现内容 |
|------|---------|
| **语音控制** | 开始/停止监听、撤销、清空、帮助 |
| **基础图形** | 圆、矩形、三角形、椭圆、直线、多边形（五/六边形）、文字 |
| **颜色** | 红、蓝、绿、黄、黑、白、紫、橙、粉、灰、青（含同义词与常见误识别「兰」） |
| **尺寸参数** | 半径、宽、高、边长、线宽 |
| **位置** | 12 种语义位置 + 数值坐标 |
| **线段端点** | 「从 A 到 B」语义位置及坐标形式 |
| **空心/实心** | 识别「空心」「边框」关键词 |
| **复合指令** | 支持「然后/接着/再」等拆解为多步顺序执行 |
| **容错** | 去除「请/帮我/画一个」等口语成分；归一化标点 |
| **延迟优化** | continuous 模式、interim 实时显示、80ms 步进队列 |
| **纯语音约束** | 禁用画布鼠标/触摸；键盘事件拦截 |
| **UI 反馈** | 状态指示灯、识别文本、执行日志、画笔位置指示器 |

### 4.2 部分实现 ⚠️

| 能力 | 现状 | 原因 |
|------|------|------|
| **自由手绘** | 引擎支持 `drawFreehand`，但语音解析未接入 | 需连续语音追踪路径点，Web Speech API 不适合实时流式坐标采集 |
| **多边形** | 仅支持正多边形，五角星为近似五边形 | 星形需要非均匀顶点算法，规则解析难以覆盖 |
| **文字绘制** | 支持简单「写XXX」，复杂标点/长文本提取不准 | 正则提取在嵌套指令中易误匹配 |
| **语音识别准确率** | 依赖浏览器云端识别 | 无法在本项目中控制模型质量，仅能通过同义词缓解 |

### 4.3 未实现 ❌

| 能力 | 原因说明 |
|------|---------|
| **图形选中/编辑/删除** | 当前为像素级画布，无图形对象模型；需引入 Scene Graph 才能支持选中与变换 |
| **图形移动/缩放/旋转** | 同上，缺少对象级状态管理 |
| **图层管理** | 架构未设计图层系统 |
| **LLM 语义理解** | 项目要求纯前端可运行，未接入外部 API；规则引擎覆盖常见场景但无法理解极度自由的自然语言 |
| **离线语音识别** | Web Speech API 依赖 Google 云端服务；本地 Whisper 需 WASM/后端，超出轻量 Web 应用范围 |
| **TTS 语音反馈** | 时间有限，UI 文字反馈已满足基本需求 |
| **笔刷/渐变/填充图案** | 优先级低，基础绘图能力优先 |
| **保存/导出作品** | 可通过语音扩展「导出」指令，但需额外文件 API 交互，暂未实现 |
| **多用户/协作** | 不在需求范围内 |

## 5. 指令示例与解析结果

| 用户语音 | 解析结果 |
|---------|---------|
| 开始监听 | `{ type: 'start' }` |
| 画一个红色的圆 | `{ type: 'draw', actions: [{ type: 'circle', params: { color: '#ef4444' } }] }` |
| 在中心画蓝色矩形宽200高100 | `{ type: 'draw', actions: [{ type: 'rectangle', params: { position: '中心', color: '#2563eb', width: 200, height: 100 } }] }` |
| 从左上到右下画一条线 | `{ type: 'draw', actions: [{ type: 'line', params: { from: {80,80}, to: {820,520} } }] }` |
| 画一个红圆然后画蓝色三角形 | 复合指令 → 2 个 actions 顺序入队 |
| 撤销 | `{ type: 'undo' }` |

## 6. 技术选型说明

### 为何选择 Web Speech API？

- 浏览器原生支持，零后端部署
- 中文识别效果在 Chrome/Edge 上可接受
- `continuous` + `interimResults` 可降低感知延迟

### 为何选择规则解析而非 LLM？

- 无网络 API 密钥依赖，开箱即用
- 响应确定、可预测，适合绘图参数提取
- 延迟低于远程 LLM 调用（通常 < 1ms 本地解析）

### 局限性

- 需联网（Speech API 云端识别）
- 仅 Chrome/Edge 完整支持
- 极度口语化或歧义指令可能解析失败

## 7. 运行方式

1. 使用 Chrome 或 Edge 打开项目
2. 因浏览器安全策略，需通过本地 HTTP 服务访问（不可直接双击 `file://` 打开）
3. 允许麦克风权限
4. 说「**开始监听**」，随后即可语音绘图

```bash
# 方式一：Python
python -m http.server 8080

# 方式二：Node.js
npx serve .
```

访问 `http://localhost:8080` 即可使用。

## 8. 后续改进建议

1. **引入图形对象模型（Scene Graph）**：使选中、移动、删除成为可能
2. **接入轻量 LLM**（如本地 Ollama）：处理自由描述「画一只简笔画小猫」
3. **离线 ASR**：集成 whisper.cpp WASM 实现无网环境
4. **语音导出**：「保存作品」→ Canvas `toDataURL` 下载
5. **指令确认机制**：对歧义指令语音反问确认，提升容错
